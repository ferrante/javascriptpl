Lista elementów a tablica

@porada #5

Obecność wielu tablico-podobnych obiektów to jedna z bolączek każdego początkującego programisty JavaScript. NodeList jest tego doskonałym przykładem.

Nie wgłębiając się w szczegóły, NodeList to lista elementów drzewa DOM — prościej mówiąc: wynik użycia metody typu document.getElementsByTagName(), document.getElementsByClassName(), document.querySelectorAll() itp.

Czym lista elementów przypomina tablicę?

Spróbujmy znaleźć wszystkie elementy <p>, których bezpośrednim rodzicem jest <body>:

var myList = document.querySelectorAll("body > p");

Używając właściwości length, możemy sprawdzić, ile ich jest:

console.log(myList.length); //→ 10
console.log(myList[5]); //→ <p class="content">…</p>

Co ją odróżnia?

Schody zaczynają się wówczas, gdy pojawia się konieczność użycia jednej z metod modyfikujących. Załóżmy, że chcielibyśmy „obciąć” ostatni element myList:

myList.pop(); //→ TypeError: 'undefined' is not a function

Spróbujmy dodać nowy element do listy:

var first = document.getElementsByClassName("first")[0];
myList.unshift(first); //→ TypeError: 'undefined' is not a function

Powód błędów? NodeList to nie Array, dlatego też oba te obiekty nie dzielą tych samych metod.

W jaki sposób przekształcić listę elementów w tablicę?

Np. używając jednego z krótszych i prostszych sposobów:

var myArray = Array.prototype.slice.call(myList, 0);

Sprawdźmy teraz możliwości naszej tablicy:

myArray.pop(); //→ <p class="footer">…</p>
console.log(myArray.length); //→ 9

var first = document.getElementsByClassName("first")[0];
myArray.unshift(first); //→ 10

JSLint Twoim przyjacielem

@varjs #4

Bardzo popularnym rozwiązaniem pozwalającym sprawdzać poprawność kodu pod względem dobrych praktyk jest JSLint autorstwa Douglasa Crockforda.

Alternatywnym projektem dla JSLint jest... JSHint.

Oba projekty można użyć w systemach continuous integration, które mogą np. sprawdzać po każdym commicie, czy kod jest poprawny.

parseInt("08")

@varjs #3

Pamiętaj, że parseInt traktuje stringa jako liczbę w systemie ósemkowym, jeśli na jego początku pojawi się "0". Dobrą praktyką jest umieszczanie zawsze "10" jako drugiego argumentu, który mówi, że powinniśmy sparsować liczbę w systemie dziesiętnym.

parseInt("08"); // 0
parseInt("08", 10); // 8

"for in" nie dla tablic

@varjs #2

Pamiętaj, że pętle for in stworzone są tylko i wyłącznie do iteracji po obiektach:

var obj = { key : 1 };
for (var i in obj) {
    console.log(i); // "key"
}

For in używany z tablicami będzie iterował również przez własności .prototype tablicy, co jest efektem niepożądanym.

Tworzenie tablic i obiektów

@varjs #1

Zapomnij o tworzeniu tablic i obiektów w, wydawałoby się, klasyczny sposób:

var arr = new Array();
var obj = new Object();

W JavaScript robi się to następująco:

var arr = [];
var obj = {};

JavaScript umożliwia tworzenie tych struktur natywnie, nie musimy dodatkowo wołać ich konstruktora. Poza tym można je nadpisać, co może przysporzyć dużo problemów:

var Object = null;
